# Тестирование API сервиса Яндекс Самокат (Sprint 7)

## Описание проекта

Этот проект содержит автоматизированные тесты для API учебного сервиса Яндекс Самокат. Тесты написаны на Python с использованием pytest и allure для генерации отчетов.

## Структура проекта

- `tests/` - директория с тестами
  - `test_courier_creation.py` - тесты создания курьера
  - `test_courier_login.py` - тесты авторизации курьера
  - `test_order_creation.py` - тесты создания заказа
  - `test_orders_list.py` - тесты получения списка заказов
  - `test_delete_courier.py` - тесты удаления курьера (дополнительно)
  - `test_accept_order.py` - тесты принятия заказа (дополнительно)
  - `test_get_order_by_number.py` - тесты получения заказа по номеру (дополнительно)
- `api.py` - клиентские классы для работы с API (CourierApi, OrdersApi) и совместимые процедурные обёртки
- `data.py` - константы (базовый URL, ручки, шаблоны данных, сообщения ошибок)
- `utils.py` - утилиты (генерация случайных строк и т.д.)
- `conftest.py` - конфигурация pytest с фикстурами (в корне проекта)
- `requirements.txt` - зависимости проекта
- `target/` - директория с результатами тестов и отчетами
  - `allure-results/` - результаты тестов для Allure
  - `allure-report/` - сгенерированный HTML отчёт

## Зависимости

Установите зависимости с помощью:

```bash
pip install -r requirements.txt
```

## Запуск тестов

### Запуск всех тестов

```bash
pytest
```

### Запуск с генерацией результатов Allure

```bash
pytest --alluredir=target/allure-results
```

### Генерация отчета Allure

```bash
allure generate target/allure-results -o target/allure-report --clean
```

### Просмотр отчета

Откройте файл `target/allure-report/index.html` в браузере.

### Запуск сервера Allure

```bash
allure serve target/allure-results
```

## Описание тестов

### Основные тесты

1. **Создание курьера** (`test_courier_creation.py`)
   - Создание курьера с валидными данными
   - Попытка создания дубликата курьера
   - Создание курьера без обязательных полей

2. **Авторизация курьера** (`test_courier_login.py`)
   - Успешная авторизация
   - Авторизация с неправильными данными
   - Авторизация без обязательных полей

3. **Создание заказа** (`test_order_creation.py`)
   - Создание заказа с разными цветами (пустой, BLACK, GREY, оба цвета)

4. **Получение списка заказов** (`test_orders_list.py`)
   - Получение списка заказов

### Дополнительные тесты

5. **Удаление курьера** (`test_delete_courier.py`)
   - Успешное удаление курьера
   - Удаление без ID
   - Удаление с несуществующим ID

6. **Принятие заказа** (`test_accept_order.py`)
   - Успешное принятие заказа
   - Принятие без ID курьера
   - Принятие с неправильным ID курьера
   - Принятие без ID заказа
   - Принятие с неправильным ID заказа

7. **Получение заказа по номеру** (`test_get_order_by_number.py`)
   - Успешное получение заказа
   - Получение без номера трека
   - Получение с неправильным номером трека

## Фикстуры

- `courier` — создаёт курьера перед тестом и удаляет его после (teardown)
- `order` — создаёт заказ и возвращает номер трека

## Требования к оценке

- Корректное именование элементов
- Клиентские классы по ресурсам API (например, CourierApi, OrdersApi)
- Проверка тела и кода ответа
- Тесты независимы
- Тестовые данные создаются перед тестом и удаляются после
